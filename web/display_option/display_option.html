Understanding the PlantUML Dependency Display option

Usage

Description

You can run PlantUML Dependency using the Display option to :

  • Filter output elements following their type and the way they are found which may be :
    • parsed abstract class (a)
    • parsed class (c)
    • parsed enums (e)
    • parsed interfaces (i)
    • not parsed class (n)
    • not parsed interfaces (v)
    • all (*)

Note that "parsed" specifies an element found as a main object in a source file.

In a later release, the Display option will probably allow to :

  • Display elements by package only
  • Filter output relations

Default

The default Display option is :

  • -d *

which is equivalent to :

  • -d aceinv

Notes

  • -d ac* will behave the same as -d *
  • -d aceinv* will behave the same as -d *
  • -d *e* will behave the same as -d *
  • -d aiaaiaaccc will behave the same as -d aci
  • -d axgh will behave the same as -d a

Examples

I would like to see :

  • All elements which are not part of the parsed source files (i.e. my project for instance) : -d nv
  • All elements except the interfaces which are not part of the parsed source files (i.e. my project for instance) and are implemented by the parsed source files : -d acein
