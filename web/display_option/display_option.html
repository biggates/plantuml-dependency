Understanding the PlantUML Dependency Display option

Description

You can run PlantUML Dependency using the Display option to :
  • Filter output elements following :
    • their type which may be :
      • class (c)
      • abstract class (a)
      • interface (i)
      • enum (e)
      • * (= all)
    • the way they are found which may be :
      • parsed (p)
      • not parsed (= just seen) (s)
  • Filter output relations which have to be displayed between displayed elements following :
    • their type which may be :
      • use (u)
      • implement (i)
      • extend (e)
      • * (= all)
    • the first element type which may be :
      • parsed [class,abstract class,interface,enum,*] (caie*)
    • the second element which may be :
      • parsed [class,interface,enum,abstract class,*] (caie*) as [static,normal,*] (sn*) import
      • not parsed [class,interface,enum,abstract class,*] (caie*) as [static,normal,*] (sn*) import
The default Display option is :

-d=[elements=*,*][relations=*,*,*,*,*,*]

which is equivalent to :

-d=[elements=caie,ps][relations=uie,caie,caie,sn,caie,sn]
